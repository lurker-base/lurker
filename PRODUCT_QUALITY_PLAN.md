# LURKER Product Quality тАФ Priority Plan
> Generated: 2026-03-17 | Focus: user-visible credibility fixes

## ЁЯФ┤ P0 тАФ Critical (Broken for ALL visitors)

### 1. тЬЕ FIXED тАФ Homepage live signals never load
- **Bug:** `docs/index.html` fetched `data/cio_feed.json` (relative) тЖТ resolves to `docs/data/cio_feed.json` which **doesn't exist** on GitHub Pages
- **Impact:** Live signals section shows "scanning..." forever then silently fails
- **Fix applied:** Changed fetch to `raw.githubusercontent.com/lurker-base/lurker/main/signals/cio_feed.json` (same pattern as live.html)

### 2. тЬЕ FIXED тАФ live.html reads 10-day stale data
- **Bug:** `live.html` fetched `signals/latest.json` тАФ last updated **March 7** (10 days ago). That file is generated by `generate_signal.py` which is a **manual** script, not auto-run.
- **Impact:** Visitors see old tokens from March 7 with stale prices
- **Fix applied:** Changed to `signals/cio_feed.json` which is auto-updated by the running scanner and auto-pushed every 15 min

### 3. тЬЕ FIXED тАФ live.html format mismatch with cio_feed
- **Bug:** `live.html` render code expected `token.score`, `token.risk.level`, `token.age_hours` but CIO feed uses `scores.cio_score`, `risk_level`, `timestamps.age_hours`
- **Impact:** Would have caused JS errors with new feed
- **Fix applied:** Added CIO Ultra format adapter in the loadData function

## ЁЯЯб P1 тАФ Credibility Killers (Fixed)

## ЁЯЯа P2 тАФ Next Sprint (NOT done yet тАФ needs implementation)

### 7. Scanner finds 0 candidates right now
- **Root cause:** `scanner_cio_ultra.py` is running (PID visible) but rejected: `low_liq: 2, low_volume: 5, known_token: 7, too_old: 16, low_tx: 1`
- **The 48h anti-relist window** in `is_new_token()` + `known_token` rejection means most Base tokens are already in the registry тЖТ rejected
- **Fix target:** In `process_candidate()`, line ~310: change known_token threshold from 48h to 168h (7 days) тАФ OR remove the known_token rejection entirely and rely on the MAX_AGE_ULTRA filter instead
- **Alternative:** Add GeckoTerminal as additional data source (currently only uses DexScreener 5 endpoints)

### 8. Badge inflation тАФ every token gets 3+ badges
- **Problem:** Current badge system: SUPER FRESH / FRESH / WHALE / GEM / BUZZ / PUMPING тАФ all look positive. No badge indicates risk or caution.
- **Fix target:** In `live.html` renderTokens(), add risk-weighted badge filtering:
  - Show max 3 badges per token
  - Risk badge always visible
  - Don't show "ЁЯФе SUPER FRESH" for tokens with `low_activity` risk tag

### 9. Category correctness тАФ "CIO" tier means nothing to users

- **Fix target:** Replace tier display with user-facing labels:

  - CERTIFIED тЖТ "VERIFIED тЬУ"
  - Add `tier_label` field in scanner output

### 10. `docs/data/signals/` directory is 13 days stale
- **Problem:** `docs/data/signals/*.json` last updated March 4 тАФ these are a dead copy
- **Fix target:** Either:

### 12. signals/latest.json is dead weight

- **Fix target:** Either remove it or have scanner_cio_ultra.py also write to `signals/latest.json` in V2 format
